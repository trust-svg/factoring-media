- 利益率目標: 20%

## 各コンポーネント仕様

### 1. アラートストリップ

ページ最上部に常時表示。問題ゼロの場合は非表示。

| アイテム | データソース | クリック先 |
|---------|------------|----------|
| 在庫切れ N件 | `items WHERE stock=0` | `/listings` |
| 未読メッセージ N件 | `messages WHERE read=false` | `/chat` |
| 価格アラート N件 | `price_alerts WHERE active=true` | `/listings?tab=pricing` |
| 同期待ち N件 | 未同期出品数 | `/listings` |

スタイル: 黄背景（警告）、赤背景（緊急 = 未読5件以上 or 在庫切れ10件以上）

### 2. 達成ボード（3カラム）

**売上達成カード:**
- 大: 現在売上額（¥X.XM）/ 目標（¥5M）
- プログレスバー: 達成率 %
- 小: 「予測: ¥X.XM」（月末着地予測）+ 目標超過/未達アイコン

**利益率カード:**
- 大: 現在利益率（XX.X%）/ 目標（20%）
- プログレスバー: 目標比 %
- 小: 前月同日比（+/-X.X%）

**利益達成カード:**
- 大: 現在利益額（¥XXX,XXX）/ 目標（¥1M）
- プログレスバー: 達成率 %
- 小: 「予測: ¥X.XM」

**ペース予測計算式:**
```
予測月末売上 = 現在売上 ÷ 経過日数 × 月の総日数
経過日数 = 今日の日付（1始まり）
※ 月初1日目（経過日数=1）はゼロ除算にならないため問題なし
※ APIは当月1日以降のみ呼ばれる前提
```

### 3. KPI前月同日比パネル（左）

前月の同日時点の累計と比較した差分・増減率を表示。

| 指標 | 今月累計 | 前月同日比 |
|------|---------|----------|
| 売上 | ¥X,XXX,XXX | +12% ↑ |
| 利益率 | 18.4% | -1.6% ↓ |
| 出品数 | 142件 | +8件 ↑ |
| 在庫切れ | 7件 | -3件 ↑（改善） |
| 注文数 | 89件 | +14% ↑ |

矢印と色（緑=改善、赤=悪化）でパターン認識を支援。在庫切れは減少が「良い」のでロジック反転。

### 4. 月別カレンダーヒートマップ（右）

- 7列グリッド（日〜土）
- 各日のセル：売上ありの日は●（売上額に比例してサイズ or 色の濃さ変化）
- 売上ゼロの日は○（空円）
- 未来の日は薄灰
- 「今日」セルに本日の売上とオーダー数をリアルタイム表示
- ホバーで日付・売上・利益・注文数のツールチップ

### 5. 30日トレンドグラフ（既存改良）

既存の ApexCharts グラフを維持しつつ：
- 目標日割りライン追加（¥5M ÷ 月日数 × 経過日数の累計線）
- グラフエリアを現状より縮小（達成ボードを優先表示）

```json
    "actual": 590000,
```

```json
{
  "year": 2026,
  "month": 4,
  "days": [
    {"date": "2026-04-01", "revenue": 120000, "orders": 3, "profit": 22000},
    {"date": "2026-04-02", "revenue": 0, "orders": 0, "profit": 0},
```

### `GET /api/overview/alerts`

```json
{
  "out_of_stock": 7,
  "unread_messages": 3,
  "price_alerts": 2,
  "sync_pending": 1,
  "severity": "warning"
}
```

`severity`: `"ok"` (全ゼロ) / `"warning"` / `"critical"` (在庫切れ10以上 or 未読5以上)
