# マッチングアプリ攻略 Bot Suite — 設計書

**作成日:** 2026-04-12  
**ステータス:** 設計確定

---

## 概要

マッチングアプリ攻略をステージ別にサポートするLINE Bot群。フリーミアムモデルで入口を広げ、Standardへの転換・Premiumへのアップセルを狙う。ZINQのメッセージAI（¥4,980）はこのSuiteに統合して廃止。既存ユーザーなし。

**ターゲット:** マッチングアプリを使用している20〜35歳の日本人男性。

## Section 1: プロダクト概要

## Section 2: Free体験設計

### 体験フロー

```
LINE Bot追加
 → 「写真を1枚送ってください」
 → AI診断（10〜20秒）
 → 結果送信（スコア＋3つの問題点のみ）
 → 自然なCTA
```

### 診断結果の出力例

```
📊 プロフィール診断結果

スコア: 6.8 / 10

改善ポイント:
① 背景に生活感が出ている（-1.2点）
② 正面顔のみで表情が硬い（-0.9点）
③ 光が逆光で顔が暗い（-0.6点）

改善すれば 8.5点 まで上げられます。
```

**「どう改善するか」の具体的な方法はStandard以上のみ提供。**

### CTAタイミング

| タイミング | 内容 |
|---|---|
| 診断直後 | 「改善写真の撮り方を知りたい方は→」 |
| 翌日（オプトイン制） | 「自己紹介文も診断できます（Standard）」 |
| 3日後（オプトイン制） | 「メッセージAIも試してみませんか？」 |

リマインドはオプトイン制。オンボーディング時に選択させる。

### Freeで出さないもの（Standard以上）

- 改善方法の具体的アドバイス
- 自己紹介文リライト
- 2回目以降の写真診断
- デート・関係構築Bot
- 月次レポート（Premium）

### プライバシー

写真本体は診断完了後に即削除。スコア・改善ポイントのテキストデータのみDBに保存し、月次レポートの変化推移に使用。ユーザーには明示する：

> 「写真は診断後即削除します。スコアデータのみ記録します（月次レポート用）」

### バイラル拡散

紹介で**お互いに1ヶ月無料**（Standard相当）。

- LINE UIDで重複管理（同一人物の複数取得を防止）
- LINEは電話番号1つにつき1アカウントが原則のため自然に制限がかかる
- Free診断は1アカウント1回限り（UID管理で制御）

## Section 3: 技術設計

### アーキテクチャ

```
LINE Platform
    ↓ Webhook
FastAPI (メインサーバー)
    ├── Bot Router（どのBotか判定）
    ├── Claude Vision API（写真診断）
    ├── Claude API（文章生成・レポート）
    └── Scheduler（月次レポート・リマインド）

PostgreSQL
    ├── users（LINE UID・プラン・紹介コード）
    ├── usage（Bot別利用回数）
    └── history（スコア・改善ポイントのテキストのみ。写真は保存しない）

Square（決済）
    └── Webhook → プラン変更をDBに反映
```

### Bot切り替え

1つのLINE Bot内でリッチメニューで切り替え。複数Bot登録は運用が煩雑なため非推奨。

```
リッチメニュー
├── 📸 プロフィール診断
├── 💬 メッセージAI
├── 📅 デートプラン
└── 💑 関係構築
```

### 利用回数管理

```
Standard: Bot別に月10回
 → 超えたら「今月の上限です。Premiumで無制限に→」
Premium: カウントしない
```

### ZINQからの流用

| 要素 | 対応 |
|---|---|
| LINE Webhook処理 | ほぼそのまま流用 |
| Claude API呼び出し | プロンプトのみ変更 |
| Square決済連携 | ZINQからそのまま流用 |
| scheduler.py | 月次レポート送信に転用 |

**新規開発は写真受信→Vision API処理とリッチメニュー設定のみ。**

### コスト管理

Claude Vision APIは1回あたり約$0.01〜0.02。Free診断を1回限りに制限することでコスト上限を制御する。
