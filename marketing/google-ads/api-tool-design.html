1. Executive Summary

TrustLink is developing an internal reporting tool that retrieves daily campaign performance data from Google Ads via the API. The tool is read-only — it does not create, modify, or delete any campaigns, ads, keywords, or account settings. Its sole purpose is to generate automated performance reports and deliver them via LINE Messaging API to internal stakeholders.

Key Points:
  • Read-only access — No campaign creation or modification
  • Internal use only — Used by company employees exclusively
  • Single account — Monitors our own Google Ads account only
  • Daily frequency — One API call per day (scheduled at 8:00 AM JST)

2. Company Overview

TrustLink is a digital marketing company based in Japan that operates mobile app promotion campaigns for dating and social networking applications targeting mature audiences (50–80 age demographic). We use Google Ads Search campaigns to drive app registrations.

3. Tool Purpose and Functionality

3.1 Problem Statement

Currently, performance data must be manually exported from the Google Ads web interface as CSV files, then processed locally. This manual workflow is time-consuming and error-prone, and prevents timely daily monitoring of campaign performance.

3.2 Solution

An automated Python script that:

  1. Connects to the Google Ads API once daily
  2. Retrieves campaign, keyword, and ad group performance metrics
  3. Analyzes the data against predefined benchmarks
  4. Generates a summary report
  5. Sends the report to internal stakeholders via LINE messaging

3.3 Supported Campaign Types

  • Search campaigns

5.2 GAQL Queries

Example query for campaign performance:

SELECT
  campaign.name,
  campaign.status,
  metrics.impressions,
  metrics.clicks,
  metrics.ctr,
  metrics.average_cpc,
  metrics.cost_micros,
  metrics.conversions,
  metrics.cost_per_conversion,
  metrics.search_impression_share
FROM campaign
WHERE segments.date = 'YYYY-MM-DD'
  AND campaign.status = 'ENABLED'

5.3 Rate Limiting Compliance

  • Maximum 3 API requests per daily execution
  • Single execution per day (8:00 AM JST)
  • No parallel requests
  • Well within Google Ads API rate limits

6. Data Handling and Security

6.1 Data Storage

  • No persistent database — data is processed in-memory and discarded after report generation
  • No Google Ads data is stored long-term
  • Reports contain only aggregated metrics (no PII)

6.2 Credential Security

  • OAuth credentials stored in local .env file
  • .env files are excluded from version control via .gitignore
  • No credentials are hardcoded in source code
  • Tool runs on a local machine (not a shared server)

6.3 Access Control

  • Tool is accessible only to company employees
  • No web interface — CLI-only execution
  • Single Google Ads account (our own)
