# Claude Workspace

This workspace contains multiple independent directories.
They are NOT related unless explicitly stated.

## Directory Structure

### client-projects
Client-specific projects.
Each project is isolated by client and contract.
Content here must NOT be reused without abstraction.

### products
My own products and tools.
Includes applications, integrations, and internal systems.
Code here may evolve into long-term assets.

### marketing
Marketing assets for my own services or products.
These are used for promotion, conversion, and experiments.
Iteration and optimization are expected.

### reusable
Generalized templates and assets intended for reuse.
No client-specific or confidential information is allowed.

### ideas
Notes, concepts, drafts, and business ideas.
This directory is documentation-focused, not production code.

## Rules for Claude

- Always work ONLY in the directory explicitly specified by the user.
- Do NOT mix logic, content, or context across directories.
- client-projects must remain isolated.
- reusable must contain abstracted, non-client-specific content only.
- If no target directory is specified, ask before proceeding.
