# Factcheck AI Cross-Check

確信度が高くても間違っていると損失が大きい事実（事業数値・法律条文・統計・API仕様・契約条件）は、1つの AI の回答で確定しない。Claude と Codex の2系統で独立に検証し、一致したものだけ採用する。不一致はユーザーに判断を委ねる。

## いつ使うか

- 法律・規制の条文（例: 弁護士法72条、特商法、景表法の条件）
- 事業の確定数値（広告予算配分、収益分岐、税率、手数料率）
- 統計・市場データの引用
- 外部 API の仕様（廃止予定・必須パラメータ・レート制限）
- 「これ間違ってたら公開物・送金・申告に響く」もの全般

## いつ使わないか

- 一般知識・可逆な作業・コードの軽微な確認（オーバーキル）
- 出典が手元にあり一次ソースを直接読めばよいとき（その場合は一次ソースが最優先）

## 手順

### 1. 1次回答（系統A: Claude）

- 主張を1文で確定形に書く（"Xは Y である"）
- 根拠ソースを必ず添える（URL・条文番号・ドキュメント箇所）。ソース無しの記憶ベース回答は「未検証」とラベル
- 不確実な箇所を明示

### 2. 独立検証（系統B: Codex）

- 系統Aの結論を**見せずに**同じ問いを投げるのが理想（バイアス回避）。最低でも「この主張は正しいか、独立に調べて」と依頼
- Codex 側にも根拠ソースを要求
- `codex-handoff` スキルの形式で問いと制約を渡すと精度が上がる

### 3. 突き合わせ

| 結果 | 判定 | アクション |
|------|------|-----------|
| 両系統が一致＋出典も整合 | **採用** | そのまま使用。出典を残す |
| 一致するが出典が弱い／古い | **保留** | 一次ソースを直接確認してから採用 |
| 不一致 | **ユーザー判断** | 両者の主張と根拠を並べて提示。勝手に多数決しない |

### 4. 記録

確定した事実と出典を、公開物・申告・送金の根拠として残す。法務系は memory の該当 Gotcha（例: `saimu-media-bengoshiho-72`）に追記。

## 出力形式

```
factcheck-ai-cross: <主張>
系統A(Claude): <結論> / 出典: <...>
系統B(Codex):  <結論> / 出典: <...>
判定: 採用 / 保留(一次ソース要確認) / ユーザー判断(不一致)
確定事実: <一致した場合の最終文＋出典>
```

不一致を「たぶんこっち」で握りつぶさない。コストの高い事実ほど、わからないことを正直に出す。
