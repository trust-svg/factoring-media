# SNSバズリサーチ＆トレンド分析

SNS上のバズコンテンツを収集・分析し、「なぜ伸びたか」の構造パターンを抽出するスキル。
分析結果は人間が読むレポート形式と、自動投稿システム（threads-auto等）に直接統合できるJSON形式の2モードで出力できる。

## 引数

```
/sns-research <キーワード> [オプション]
```

| 引数 | 説明 | デフォルト |
|------|------|-----------|
| `<キーワード>` | 検索ジャンル/キーワード（必須） | — |
| `--platform` | 対象プラットフォーム（カンマ区切り） | `threads,x,youtube` |
| `--mode` | 出力形式: `report` or `directive` | `report` |
| `--competitors` | 競合アカウント（カンマ区切り） | なし |
| `--days` | 振り返り期間（日数） | `7` |
| `--limit` | プラットフォームあたりの最大取得数 | `30` |

### 使用例
- `/sns-research 転職` — 転職ジャンルのバズ分析レポート
- `/sns-research 転職 --mode directive` — 自動投稿システム用JSON
- `/sns-research 美容 --platform threads,x --days 14`
- `/sns-research 転職 --competitors @career_hack,@tenshoku_pro`

## 実行手順

以下の5レイヤーを順番に実行する。各レイヤーは前のレイヤーの出力を入力として使う。

---

### Layer 1: 収集（Collection）

WebSearchとWebFetchを使って、各プラットフォームからバズ投稿を収集する。
API直接アクセスは不要 — Web検索経由で十分な情報が取れる。

#### X (Twitter) の収集

以下のような検索クエリをWebSearchで実行:
- `site:x.com "{キーワード}" lang:ja` — 直近の人気投稿
- `{キーワード} バズ ツイート site:togetter.com OR site:x.com` — まとめ経由
- `{キーワード} min_faves:100 lang:ja site:x.com` — いいね100以上

各結果URLをWebFetchで取得し、投稿テキスト・いいね数・RT数・リプライ数を抽出する。
X.comのURLが取得できない場合は、まとめサイトやニュースサイト経由で間接的に取得する。

#### Threads の収集

- `site:threads.net "{キーワード}"` でWebSearch
- `{キーワード} threads バズ 投稿` で間接検索
- 結果URLをWebFetchで内容取得

#### YouTube の収集

- `{キーワード} site:youtube.com` でWebSearch（直近の人気動画）
- `{キーワード} YouTube 人気 最新` で間接検索
- 動画タイトル、概要文、再生回数、いいね数を取得
- YouTubeは「ネタ元」としての収集が主目的（投稿テンプレ分析はX/Threads中心）

#### 収集結果の構造化

各投稿を以下の形式で保存:
```json
{
  "platform": "x",
  "url": "https://x.com/...",
  "author": "@handle",
  "text": "投稿本文",
  "engagement": {
    "likes": 1500,
    "replies": 230,
    "shares": 450
  },
  "date": "2026-03-25",
  "source_type": "direct"
}
```

注意: 正確な数値が取得できない場合は推定値を使い、`"estimated": true` フラグを付ける。
取得できなかった項目は `null` にする。嘘の数字を捏造しない。

---

### Layer 2: 構造分析（Structure Analysis）

収集した各投稿に対して、以下の観点で構造を分解する。
これは収集したテキストを読んでCodex自身が分析する作業であり、追加のツール呼び出しは不要。

#### 分析項目

| フィールド | 説明 | 値の例 |
|-----------|------|--------|
| `hook_type` | フックの分類 | 暴露系, 断言系, リスト系, 質問系, コメント誘導系, 逆説系, 数字インパクト系, ストーリー系 |
| `hook_line` | 1行目（注意を引くフレーズ） | 「転職エージェントが絶対教えない事実」 |
| `structure` | 投稿の論理構造 | フック → 常識否定 → 具体例 → 行動喚起 |
| `emotion` | 狙っている感情 | 怒り, 共感, 驚き, 憧れ, 好奇心, 不安, 希望 |
| `char_count` | 文字数 | 198 |
| `engagement_rate` | エンゲージメント率（推定可） | 4.2 |
| `copyable_pattern` | 抽象化したテンプレート | 「○○が絶対教えない△△」→ 常識否定 → 3具体例 → 「知らないと損」で締め |

#### hook_type の分類基準

投稿の1行目を読んで、以下のどれに最も近いかを判定:

- **暴露系**: 「○○の裏側」「○○が教えない真実」「実は○○」
- **断言系**: 「○○は間違い」「○○するな」「○○一択」
- **リスト系**: 「○選」「○つの理由」「○パターン」
- **質問系**: 「○○って知ってる？」「なぜ○○なの？」
- **コメント誘導系**: 「答えはコメント欄」「どっち派？」
- **逆説系**: 「○○なのに△△」「実は逆効果」
- **数字インパクト系**: 「月収○万→○万」「○%の人が知らない」
- **ストーリー系**: 「3年前の僕は○○だった」「あの日から変わった」

---

### Layer 3: トレンド検出（Trend Detection）

構造分析の結果を集計し、以下のトレンドを検出する。

#### 検出方法

1. **テーマ集計**: 収集した投稿のテーマ（何についての投稿か）をタグ付け
2. **hook_type集計**: どのフックタイプが多いか、どれが高エンゲージメントか
3. **上昇/下降判定**:
   - 直近3日のエンゲージメント平均が7日平均より高い → 上昇トレンド
   - 直近3日が7日平均より低い → 下降トレンド
   - データが少ない場合は、投稿の鮮度と反応の強さから定性的に判断してもよい

#### 出力形式

```json
{
  "trending_up": [
    {"theme": "面接裏側暴露", "signal": "直近3日でいいね平均2.3倍"},
    {"theme": "年収交渉テク", "signal": "新規投稿が急増"}
  ],
  "trending_down": [
    {"theme": "エージェント批判", "signal": "同テーマの反応率が半減"}
  ],
  "stable": [
    {"theme": "リスト系ノウハウ", "signal": "安定して高エンゲージメント"}
  ]
}
```

---

### Layer 4: 競合ウォッチ（Competitor Watch）

`--competitors` オプションが指定された場合のみ実行。

#### 手順

1. 各競合アカウントのプロフィールページをWebFetchで取得
2. 直近の投稿一覧からエンゲージメントが高いものを抽出
3. Layer 2と同じ構造分析を適用
4. 競合ごとに「得意パターン」「最近の傾向変化」をまとめる

#### 出力形式

```json
{
  "competitors": [
    {
      "handle": "@career_hack",
      "follower_count": "推定15K",
      "top_patterns": ["暴露系+具体数字", "コメント誘導型"],
      "recent_shift": "最近リスト系が増加傾向",
      "best_post": {
        "text": "...",
        "engagement": {"likes": 2300}
      }
    }
  ]
}
```

### Layer 5: 出力生成

#### report モード（デフォルト）

人間が読むMarkdownレポートを生成する。

```markdown
# SNSバズリサーチレポート: {キーワード}
> 期間: {開始日} 〜 {終了日} | 対象: {platforms} | 収集数: {total}件

## エグゼクティブサマリー
- 今週のバズ傾向を3行で要約
- 最も伸びているパターン
- 注意すべき下降トレンド

## トレンド分析
### 上昇中のテーマ
### 下降中のテーマ
### 安定テーマ

## バズ投稿 TOP10
（エンゲージメント順にランキング。各投稿の構造分析付き）

## フックライン集
（今週効いているフックラインをカテゴリ別に整理）

## 推奨アクション
（次のバッチで何をすべきかの具体的な提案）

## 競合分析（指定時のみ）
```

## 品質ガイドライン

### やるべきこと
- **構造を学ぶ**。テキストをコピーするのではなく、「なぜ伸びたか」のパターンを抽出する
- **数字は正直に**。取得できない数値は `null` か `"estimated": true` で明示する
- **ジャンル非依存**。転職でも美容でもプログラミングでも同じ手順で動く
- **鮮度重視**。古い投稿（指定期間外）は無視する

### やってはいけないこと
- バズ投稿の文面をそのままコピーして「使え」と指示すること（著作権+独自性の問題）
- 取得できなかったエンゲージメント数を適当にでっち上げること
- 1つのプラットフォームだけで判断すること（クロスプラットフォームで確認）

## 自動投稿システムとの統合

`directive` モードの出力は、threads-autoのリサーチャーエージェントから直接呼び出せる設計。
出力JSONを `state/research_directive.json` に保存すれば、ライターが次のバッチ生成時に自動で参照する。

毎朝のcronでリサーチャーがこのスキルを `--mode directive` で実行し、結果をライターに渡すフローが理想的。
